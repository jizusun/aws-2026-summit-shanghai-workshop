# Phase 2: 创建并部署 Agent

**形式**: 动手实验 | **时长**: 30 分钟

![](/images/progress-p2.zh.png)

---

## 这一步在做什么

生产级 Agent 的第一项纪律,是**从明确的范围开始**(BP1)——先回答"要解决什么问题",而不是"Agent 能做什么"。本阶段我们把范围收敛到一个具体场景:让 HR Agent 连接企业已有的政策知识、接入 HR 操作(查询余额、提交申请)、并在明确的行为规范下运行。

接下来 30 分钟,你用 AgentCore CLI 完成这一切——纯配置驱动,**没有编排代码**。这是 Managed Harness 在工程上的实际价值:迭代以分钟计,后面 Phase 5 改 Prompt 重新部署评估,你会再次感受到。

完成本阶段,你会得到一个部署在 VPC 内、可对话的企业 HR Agent。但请记住:**此刻它只是一个能运行的原型,答得好不好要等 Phase 4 的评估来客观回答**。这也是为什么我们从第一步起就埋好可观测性(BP2)——为后续的评估准备数据。

## 步骤概览

| 步骤 | 操作 | 目的 |
| --- | --- | --- |
| 2.1 | 连接并验证环境 | 确认工具链就绪 |
| 2.2 | 创建知识库 | 构建 Agent 回答问题的知识来源（对应 BP3：清晰的工具策略） |
| 2.3 | 创建 Gateway | 建立 Agent 调用外部工具的受控通道（对应 BP3） |
| 2.4 | 配置 Skills | 按需加载深度分析能力（对应 BP5：用工具替代内部推理） |
| 2.5 | 创建 Harness 并部署 | 将以上组件整合为完整 Agent，同时开启可观测性（对应 BP2） |
